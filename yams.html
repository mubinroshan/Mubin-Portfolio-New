YAMS | YNH Asset Management System

A modern IT Inventory and Network Mapping system built for YNH Hospital.

Technical Stack

Frontend Next.js 14, React 18, Tailwind CSS, ShadCN UI
Backend Firebase Firestore, Firebase Authentication
Tools Lucide Icons, QR Generator, Hardware Diagnostics

Exporting & Handover

To download the source code from Firebase Studio for local server deployment:

  1. Locate the Header: Look at the top black bar of the interface (where it says YNH Asset Matrix).
  2. Click Download: To the right of the project name, click the Download Source icon (downward arrow or ZIP folder).
  3. Extract: Save the .zip file and extract it to your directory.

Local Offline Hosting

YAMS is designed to run within a hospital LAN without requiring an active internet connection.

Prerequisites

  • Node.js 18.x or 20.x
  • Java Runtime Environment (JRE) 11+
  • Firebase CLI (npm install -g firebase-tools)

Deployment Steps

# 1. Install Dependencies
npm install

# 2. Build the Application
npm run build

# 3. Configure Local Backend
firebase init emulators
# Select: Firestore, Authentication

# 4. Start the System
firebase emulators:start

LAN Access Configuration

  • Set a static IP for the server (e.g., 192.168.1.50).
  • Update the host property in firebase.json to "0.0.0.0".
  • Open ports 3000, 8080, and 9099 in the firewall.

Folder Structure

/src/app      # Next.js App Router routes
/src/components # UI & Business components
/src/firebase # Backend config & data hooks
/src/lib      # IP calculations & Utility functions
/src/context  # Global state (Security PINs)

Key Features

  • Asset Registry: Hardware tracking with auto-generated serials (XXYYZZZ).
  • Network Map: Visualization of IP pools and conflict assignments.
  • Keyboard Tester: Diagnostic tool for rapid hardware validation.
  • Audit Reports: Filterable and printable inventory lists.
